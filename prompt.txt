Tu travailles sur le projet AUDITSCANIA.

Objectif : nettoyer et préparer le projet pour un push propre sur GitHub.

Lis ces fichiers avant de commencer :
- .gitignore (s'il existe)
- .env (s'il existe)
- docker-compose.yml
- backend/requirements.txt
- frontend/package.json

Résume ce que tu trouves. Attendre confirmation.

Applique ces tâches dans l'ordre :

1. .gitignore — Créer ou compléter
   S'assurer que ces éléments sont exclus :
   .env
   .env.*
   !.env.example
   __pycache__/
   *.pyc
   *.pyo
   .pytest_cache/
   node_modules/
   .next/
   *.log
   postgres_data/
   redis_data/
   nuclei_templates/
   trivy_db/
   flower.db
   *.db
   .DS_Store
   Thumbs.db

2. .env.example — Créer un fichier exemple
   Créer .env.example avec toutes les variables
   nécessaires mais sans aucune vraie valeur.
   Format :
     POSTGRES_USER=
     POSTGRES_PASSWORD=
     POSTGRES_DB=
     REDIS_URL=
     SECRET_KEY=
     VIRUSTOTAL_API_KEY=
     SHODAN_API_KEY=
     ABUSEIPDB_API_KEY=
     ANTHROPIC_API_KEY=
     GEMINI_API_KEY=
   Ne jamais mettre de vraies clés.

3. Vérifier qu'aucune clé API n'est en dur dans le code
   Scanner tous les fichiers Python et TypeScript pour
   détecter des patterns de type :
     AIzaSy...  (Gemini)
     sk-ant-... (Anthropic)
     any string ressemblant à une clé API
   Signaler chaque occurrence trouvée sans les corriger
   automatiquement. Attendre confirmation avant de modifier.

4. README.md — Créer un README professionnel
   Sections à inclure :
   - Titre + description courte du projet
   - Stack technique (liste simple)
   - Prérequis (Docker, Docker Compose)
   - Installation en 3 étapes :
       git clone ...
       cp .env.example .env
       docker compose up --build
   - Usage (Lab Mode / Active Mode)
   - Structure des dossiers (arbre simple)
   - Licence MIT

5. Vérifier la structure des dossiers
   S'assurer que chaque microservice a bien :
   - Un Dockerfile
   - Un requirements.txt ou go.mod
   - Aucun fichier temporaire ou de log

Attendre confirmation après chaque étape.
Ne jamais supprimer de fichiers sans confirmation explicite.
Ne jamais modifier les vraies clés si trouvées — juste signaler.
